# Deduplication command

We want to check if source dir files exist in 

Create command `kuferek dedup [--write|-w] <master-dir>...`.

Source-dir:
 - is always the current directory (`.`); not passed as an argument.
 - must not be a master-dir or a master subdir - check path.

Master-dir:
 - one or more may be given, so shell wildcards work (e.g. `202[1234]`, `/backups/*`); a source file matches if it exists in any master.
 - is read-only can't be written ever!.

Assumptions:
 - this is simple independennt process which doesn't use xattrs or any other initialization with other kuferek commands
 - `-w` or `--write` indicate write-mode, lack of param is dryrun-mode, write only to source dir
 - don't write any content into files!
 - don't delete any files, but move is allowed under process conditions
 - you can reuse existing app routines if available

Process:
 - In dryrun-mode print "Dry-run compare {source-dir} with {master-dir}" on program start.
 - check if source has files, stop if not, ignore dirs and "./.."
 - check if master has dirs, can be deep
 - read files list in source dir recursively (skipping its own `!found-in-master` subdir), remember sizes to files relation (that will be used for 1st stage mapping)
 - skip files smaller than 1024 bytes
 - skip ignored junk files by name, case-insensitive (currently `Thumbs.db`)
 - a checksum match additionally requires the file extension to match (case-insensitive), e.g. `.JPG` == `.jpg`
 - scan master dir and subdirs to find any files matching sizes of source dir
 - during scan show progress by printing currently processed directory in terminal line "Master scanning (1st): {xyz} (found {a} potential matches)" (don't append, use inplace update)
 - 2nd stage now we need to compare files using checksums, only then we ensure any source-dir file exists in master-dir stucture
 - if there there is checksum match then remember the source file matching on the list (keep some link to master), print matching to screen
   "Found: {source-match-file} <-> {master-match-file}"
 - Master may have multiple copies of the {source-match-file}, show them too but with:
   "Found: {source-match-file} <-> {another-master-match-file} (duplicate in master)
 - If a source file shares a name and size with a master file but the checksum differs (possible bitrot), print
   "Found: {source-file} <-> {master-file} - checksum mismatch!" and count it for the summary; do not move it.
 - After all scan, in write-mode create source-dir `!found-in-master` (if doesn't exist) and move all {source-match-file} into it, preserving each file's relative path.
 - Ensure file dates are intact during move
 - Sumarize move with how many files got moved and how much storage was deduped (human readable), in dryrun-mode just summarize how much can be reclaimed.
